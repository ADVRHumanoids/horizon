class horizon.solvers.ilqr.iterativeLQR(x, u, xdot, dt, N, intermediate_cost, final_cost, intermediate_constraints={}, final_constraint=None, sym_t=<sphinx.ext.autodoc.importer._MockObject object>, opt={})¶

Bases: object

The iterativeLQR class solves a nonlinear, (equality)constrained iLQR problem for a given
  • system dynamics (continuous time)
  • intermediate cost l(x,u)
  • final cost lf(x)

Equality constraints are treated by mean of projection: cost function is projected onto equality constraints.

horizon.solvers.sqp.sqpsol(name, qp_solver, problem_dict, options_dict)¶

sqpsol creates a sqp solver using Gauss-Newton approximation of the Hessian :param name: name of the solver (not used at the moment) :param qp_solver: internal qp solver name :param problem_dict: {‘f’: cost_function (residual!), ‘g’: constraints, ‘x’: variables} :param options_dict: {‘max_iter’: iterations}

Returns: a sqp object

NOTE: the cost function has to be passed in a residual form: [f1, f2, f3, …] where F = ||f1|| + ||f2|| + …
